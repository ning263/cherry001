## What This Project Is

This project explores an AI-assisted way for users to encounter, re-encounter, and discuss literary works.

The core product direction is not "AI summarizes books for users." The stronger direction is:

- A work enters into dialogue with the user.
- The user experiences life through story.
- AI helps the user meet a work at the right moment.
- AI helps the user return to a work with new life experience.

## Current Stage

The project is in early concept and content-engine design.

The immediate priority is to clarify:

- What kind of literary experience the product should create.
- What AI should and should not do.
- How to turn a work into scenes, causality, emotion, and discussion.
- What the MVP should prove before building a large product surface.

## Core Belief

AI should not replace reading. It should help users approach, experience, remember, and discuss works.

The product should protect the openness of literature. It can offer interpretations, but it should not collapse a work into one fixed conclusion.

## Product Boundary

This project should avoid becoming:

- A generic book summary tool.
- A list of abstract opinions about famous works.
- A shortcut that claims to replace reading.
- A system that over-explains works before users have experienced them.

The project should move toward:

- Story-first narration.
- Causal and emotional continuity.
- Concrete scenes instead of abstract claims.
- Discussion after experience.
- Clear separation between textual facts and interpretation.

## Collaboration Method

The user may discuss ideas during the day in a mobile ChatGPT Project. The evening development environment is this local repository.

Workflow:

1. Daytime discussion happens in ChatGPT.
2. The daytime discussion is summarized as a handoff.
3. Codex writes confirmed direction into repository docs.
4. Implementation decisions and code changes are based on repository docs.
5. GitHub private repository stores the long-term project memory.
